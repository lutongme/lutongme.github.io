Hexo从0开始（一）

#关于此贴简介

其实很早就开始做博客了，因为我的三分热度，做好几乎是弃坑状态，前一段时间电脑分区格式化什么的，把东西全弄丢了(没东西的)懒得恢复，不如做个从0开始的博客教程。。
本文讲解：安装node.js，安装git，github简单入门，搭建hexo

###开整

####node.js
下载需要用到的环境：node.js，git
先安装node.js https://nodejs.org/
选择 xxxx LTS Recommended For Most Users
一路next，然后install ，中途可能会需要申请管理员权限，同意就行

####安装git
下载git https://www.git-scm.com/download/win
点击安装，第一页应该是使用协议等信息，next到下一步
第二步，选择安装位置。这里提示安装途径，默认即可
第三步，装组件。下一步是安装组件，作者也是个英语渣，本文只讲hexo安装教程，不做其他用途，所以按照提示走就行了，默认即可
第四步，创建开始菜单目录 这一步是提示安装到菜单文件夹，默认即可
选择Git使用的默认编辑器 这一步是提示使用什么作为git的文本编辑器，上面默认是使用vim编辑器，那就默认把
配置path环境 这一步是三选一，作者从网上听取建议，第二项是在Windows命令符下可以使用git，这个选项被认为是安全的，因为它只向PATH添加一些最小的 Git包，以避免使用可选的Unix工具混淆环境。，您将能够从 Git Bash 和 Windows 命令提示符中使用 Git，这里选择第二项即可。
使用 OpenSSL 库 这一步选择openssl扩展把，传输协议，第一个为https协议，第二个为ssl协议。
配置行结束转换，选第一个。
使用迷你tty
最后一步是凭证什么的，默认即可，点install

###配置git
在桌面右键，会多出2个功能，GitGUIHere和GitBashHere，这里点击GitBashHere打开页面，具体怎么叫不清楚，看起来就是一个命令行，估计就是个命令行。
输入

测试是否成功：
‘’’
ssh -T git@github.com //测试是否成功：
然后会出现一些选项，回复yes即可，会提示你：
Hi lutongme! You’ve successfully authenticated, but GitHub does not provide shell access.
//当收到此消息后，可用证明与git和github链接成功
‘’’

下一步就是给自己设置个名字，就是让Github是你上传的

###配置

打开git工具：右键桌面，点击Git Bash here，打开

安装hexo:

然后创建个用来存储博客的地方，你可以用任何一个方法，手动也行，命令也行
手动比较简单，这里简单喷几句用命令行的方法，我的博客打算存储在d盘，那么我先cd / (切换盘啥懒得记) 然后cd d (前往d盘)，init blog //创建博客的文件夹，cd blog就行了
接下来安装hexo到文件夹内：hexo init，安装完成后
使用命令；

这个时候，你可以打开浏览器，访问http://localhost:4000/，如果显示出页面就是成功了
退出请使用Ctrl+c退出，ctrl和c键一起按

###创建库
打开网站：https://github.com/new
存储库名称为：用户名.github.io
然后创建存储库即可
进入存储库后，点击绿色按钮:Clone or Download
复制文本条中的网址，格式为：https://github.com/用户名/存储库名.git
